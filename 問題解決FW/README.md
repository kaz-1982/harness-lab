# 問題解決ワークスペース v01

このフォルダは、**抱えている問題を、Claude Code と対話しながら「原因」と「対応策」に整理する**ためのワークスペースです\
MECE（モレなく・ダブりなく）なロジックツリーを一緒に作りながら、問題を解決していきます

## 使い方

1. このフォルダ全体を**コピー**して、相談したい問題ごとのフォルダにする
2. フォルダ名を分かりやすいものに変える（例：`離職率の問題`）
3. Claude Code でこのフォルダを開く
4. 「**問題解決プロセスを始めて**」と伝える
5. あとは Claude の質問に答え、出てきたツリーにコメントしていくだけ

## 進む流れ

```
① 問題を整理   →   ② 原因ツリー(Why)   →   ③ 対応策ツリー(How)   →   ④ アクションプラン
   ヒアリング        なぜ起きているか           どう解決するか              誰が・いつ・何を
```

各ステップで、Claude が作ったロジックツリー（drawio 図）を一緒に見ながら、
**納得いくまでコメント → 改善**を繰り返します。

## フォルダ構成

```
.
├── CLAUDE.md          # Claude が最初に読む進行表（さわらなくてOK）
├── README.md          # このファイル
├── WORKFLOW.md        # 全体の流れの図解（WORKFLOW.drawio へのリンク＋テキスト版）
├── WORKFLOW.drawio    # 全体フロー図（drawio）
├── ChangeLog.md       # 変更履歴
├── input/             # 参考資料を入れる場所（任意）
├── output/            # 成果物がここに出る
└── harness/           # Claude 用の手順書（さわらなくてOK）
```

## 出力されるもの

`output/` に、以下が Markdown（ツリーは drawio 図ファイル付き）で生成されます。

- `01_問題定義.md` — 問題の整理
- `02_原因ツリー_why.md` ＋ `02_原因ツリー_why.drawio` — 原因の分析（全原因を等価に洗い出し）
- `03_対応策ツリー_how.md` ＋ `03_対応策ツリー_how.drawio` — 対応策の検討
- `04_アクションプラン.md` ＋ `04_優先順位マトリクス.drawio` — 実行計画

> **drawio 図の見方**：`.drawio` ファイルは [draw.io デスクトップ](https://www.drawio.com/)・[diagrams.net（ブラウザ）](https://app.diagrams.net/)・
> VS Code の「Draw.io Integration」拡張で開けます。図を開いて **「配置 → レイアウト → 水平ツリー」** を押すと自動で整列します。
> 各 Markdown にはツリー構造の**箇条書き**も併記されるので、drawio を開かなくても内容は追えます。

## 注意

- `CLAUDE.md` と `harness/` は触らないでください（Claude の動作に影響します）。
- 生成物は `output/` に出ます。
- 1フォルダ＝1つの問題、で使うのがおすすめです。
