# 対応策ツリー（How・1対1 → 必要に応じて収束）

> ③ 「どう解決するか」の検討。`harness/02_how_tree.md` の手順で作成。
> **まず原因の数だけ対策を作り（原因 Cn ↔ 対策 Mn の1対1）**、そこから**必要に応じて右へ収束**（T）させ本筋（A）へ。無理に収束しない。
> 識別番号：原因 `C#`／対策 `M#`／収束 `T#`／本筋 `A#`。drawio にも表示する。
> ユーザーと一緒に改善するたびに、このファイルを上書き更新する。

## 対象の末端原因（全件・C番号）

（`02_原因ツリー_why.md` の「根本原因の一覧（全件）」を C番号つきで再掲。**一部だけでなく全部**。これを③の起点＝左端にする。）

## 原因 ↔ 対策（1対1の対応・必ず原因の数だけ）

> **すべての原因 Cn に対策 Mn を1つずつ**付ける（番号をそろえる）。**ユーザーがこの対応を見て、対策が原因に効くか人の目で判断する。**
> 対策の数＝原因の数。ここではまだ束ねない。

| 原因（②） | 対策（1対1） | 状態(事実/仮説) |
|---|---|---|
| C1 （…） | M1 （…） | |
| C2 （…） | M2 （…） | |
| C3 （…） | M3 （…） | |
| … | … | |

## 収束（必要に応じて束ねる・無理にしない）

> 上の対策 M のうち、**まとめられるものだけ**を収束対策 T に束ねる。**束ねないものは単独のまま残してよい**（原因の数だけ対策があってもよい）。

| 収束対策 | 束ねた対策（M） | 効く原因（C） |
|---|---|---|
| T1 （束ね） | M1 + M2 + M3 | C1・C2・C3 |
| T2 （束ね） | M4 + M5 | C4・C5 |
| T3 （単独） | M6 | C6 |

## 本筋アクション（いちばん右・最少・実行する打ち手）

> 収束対策 T を寄せた「結局これをやる」。1〜数個。収束が進まなければ T＝A でよい。

| 本筋 | 束ねた収束対策（T） | 効く原因（C） |
|---|---|---|
| A1 （…） | T1 + T2 + T3 | C1〜C6 |

## 収束の流れ（図と同じ内容を箇条書きでも）

> 図：[03_対応策ツリー_how.drawio](03_対応策ツリー_how.drawio)（drawio / diagrams.net で開く）
> 原因C と1対1の対策M は白・横一直線、束ねた収束対策T・本筋A は緑。座標は `_format.md`「収束させる図の座標」で計算済み。

- C1 → M1 ┐
- C2 → M2 ┼→ T1 ─┐
- C3 → M3 ┘        │
- C4 → M4 ┐        ├→ A1
- C5 → M5 ┴→ T2 ──┤

（左：原因の数＝対策の数 → 右：必要な分だけ収束。**対策のない原因は残さない**＝カバレッジ維持。）
