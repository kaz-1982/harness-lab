# WORKFLOW.md — 全体の流れ

> 図：[WORKFLOW.drawio](WORKFLOW.drawio)（drawio / diagrams.net で開く）

全体の流れ（上から下へ。②③は納得いくまで改善ループ）：

```
困りごと・相談
  ↓
① ヒアリング（問題を整理する） → 01_問題定義.md
  ↓
② 原因ツリー Why（なぜ起きているか）
  ├─ ツリーに納得?  ──「コメントして改善」→ ② に戻る
  └─「根本原因を全件洗い出す（等価・捨てない）」→ 02_原因ツリー_why.md（＋.drawio）
  ↓
③ 対応策ツリー How（どう解決するか）
  ├─ ツリーに納得?  ──「コメントして改善」→ ③ に戻る
  └─「対応策を確定」→ 03_対応策ツリー_how.md（＋.drawio）
  ↓
④ 優先順位づけ・アクションプラン → 04_アクションプラン.md（＋優先順位マトリクス.drawio）
  ↓
実行へ
```

## 各ステップの担当ファイル

| ステップ | 担当ファイル | 出力 |
|---------|------------|------|
| ① ヒアリング・問題定義 | `harness/00_intake.md` | `output/01_問題定義.md` |
| ② 原因追及（Why）ツリー | `harness/01_why_tree.md` | `output/02_原因ツリー_why.md` |
| ③ 対応策（How）ツリー | `harness/02_how_tree.md` | `output/03_対応策ツリー_how.md` |
| ④ アクションプラン | `harness/03_action_plan.md` | `output/04_アクションプラン.md` |
| （全ステップ共通の原則） | `harness/_mece_guide.md` | — |

## ポイント

- **②と③は「作って終わり」ではなく、ユーザーと一緒に改善するループ**です（図の戻り矢印）。
- 原因を**出し切ってから**（全件・等価に、捨てずに）対応策に進みます。挙げた原因すべてに対応策をつけ（**1施策で複数原因に効いてよい**）、本筋まで引き上げて、品質は市販品同等以上（POC不可）を狙います。原因が曖昧なまま対策を考えない。
- 各ツリーは **MECE（モレなく・ダブりなく）** を意識して作ります。

以上。
