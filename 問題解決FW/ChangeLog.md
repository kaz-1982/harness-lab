## [0.4.1] - 2026-06-22

### Changed
- **③対応策ツリーを「1対1 → 必要に応じて収束」型に作り替え**（`harness/02_how_tree.md` / `templates/03_対応策ツリー_how.md` / `templates/_format.md` / `CLAUDE.md`）
  - **対策は必ず「原因の数だけ」作る（1原因1対策）**。すべての末端原因の真横に対策を1つずつ置き（横一直線）、**原因↔対策の対応をユーザーが目で見て判断**できる形にする（従来は1原因から対応策が枝分かれして増える形だった）。これでカバレッジも自動的に満たす
  - **そこから必要に応じて右へ収束**させる（まとめられる対策だけ束ねて数を減らす→本筋アクションへ）。**無理に収束しない＝原因の数だけ対策が残ってもよい**。減るほど実行しやすい
  - 起点は②の**末端原因（全件）**。収束しても原因は捨てない（カバレッジ維持）。②（右へ広がる）と③（1対1から必要分だけ狭まる）を合わせて全体像にする
  - `_format.md` の「収束させる図の座標」を更新（原因と1対1の対策は同じ y＝横一直線／収束対策・本筋は束ねる群の中央／収束は必要なところだけ）
- **原因・対策に識別番号を導入し、drawio のラベルにも表示**（`01_why_tree.md` / `02_how_tree.md` / `templates/02・03` / `_format.md` / `CLAUDE.md`）
  - 末端原因＝`C1,C2…`（②で採番）／対策＝`M1,M2…`（`Cn↔Mn` で1対1）／収束＝`T#`（束ねた M を併記）／本筋＝`A#`（束ねた T を併記）
  - ②③で同じ C番号を使い、`Cn→Mn→Tk→An` と原因↔対策の対応を番号でたどれるようにした。例の drawio も全ノードに番号を表示

## [0.4.0] - 2026-06-22

### Changed
- **②原因ツリー：「重点」マークを廃止し、全原因を等価に扱う方式に変更**
  - 一部の原因を「重点」として選ぶ仕組み（実運用で“重点を数個に絞ってそれを起点に考える”挙動を生んでいた）を全廃。末端の原因はすべて同じ重みで③に渡す
  - drawio の原因ノードはすべて通常スタイル（白）で描く。赤による重点強調をやめた（強調＝事実上の絞り込みになるため）
  - 用語・列を整理：「根本原因の一覧（全件）」から重点列を削除、「重点原因」節を削除
- **③対応策ツリー：「1施策で複数原因を解決」「解決策のレベルを上げて本筋を出す」を明文化**
  - 1つの施策が複数の原因に効くなら束ねてよい（カバレッジ表を複数原因対応に変更）。収束は任意（数を絞ること自体は目的にしない）
  - 個別施策の羅列で止めず、狙いを引き上げて「実際にとる本筋のアプローチ」を必ず1つ以上立てるステップを追加
  - 緑強調の意味を「重点原因に効く」→「複数原因に効く束ね施策・本筋のアプローチ」に再定義
- **④アクションプラン：優先順位を「効果×実現性」から「効果（インパクト）一本」に変更（QCDのQ最優先）**
  - 「実現性（工数・期間・コスト＝C/D）」軸を廃止。コスト・納期で順位を決めない・施策を捨てない（実装力は Claude が支える前提）
  - 4象限マトリクスを廃し、品質ゲート通過後に「効果 大／中／小」の帯で着手順を示す drawio に変更
  - 「最優先1〜3個に集中」の強制的な絞り込みをやめ、全施策を実行対象として残す（着手順だけ効果で決める）
- **品質基準に「POC不可・市場で十分に通用する品質を完了の定義」を全所で明記**（`CLAUDE.md` / `harness/*` / `templates/*`）

### Removed
- ②の「重点原因」マーク／選別ステップと、それに紐づく赤・緑の強調定義（`_format.md`）
- ④の「効果×実現性」2軸マトリクスと「実現性」「コスト」評価列、「最優先1〜3個に集中」の絞り込み

## [0.3.0] - 2026-06-22

### Changed
- **②原因ツリー：真因の絞り込みを廃止し、根本原因を「全件」洗い出す方式に変更**
  - 「最も効きそうな真因を1〜3個に絞る」をやめ、末端の原因をすべて残す。特に効きそうな原因は「重点」としてマーク（赤）するだけで、捨てない
  - 用語を「真因」→「根本原因（全件）＋重点原因」に統一（`CLAUDE.md` / `harness/*` / `templates/*` / `WORKFLOW.*` / `README.md` / `output/README.md`）
- **③対応策ツリー：②で挙げた全原因を対象に対応策をつける方式に変更**
  - 一部の原因だけでなく全根本原因に対応策を紐づける。「原因×対応策のカバレッジ表」を追加し、対応策のない原因（モレ）を検出する
- **③④：品質重視へ転換。「クイックウィン（すぐ作れる）」の強調を廃止**
  - 各対応策に「目指す品質水準」を必須化。既定の品質バーを「市販・既存サービスと同等以上」に設定
  - ④に「品質ゲート」を新設（品質を満たさない案は作り込むか計画に回す。品質はトレードオフ軸にしない）。「実現性」を「必要な品質を満たしたうえでの実行しやすさ」に再定義
  - 優先順位マトリクスの「クイックウィン」象限を「余力で実施」に改称
- **drawio のレイアウト崩れを修正（生成時点で整列）**
  - `templates/_format.md` の座標ルールを、ボトムアップのツリーレイアウト（葉を上から積み、親は子群の中央に置く）に全面改訂。手動の「水平ツリー」整列に依存しない
  - エッジに接続点固定（親の右中央 `exitX=1;exitY=0.5` → 子の左中央 `entryX=0;entryY=0.5`）を必須化し、矢印の斜め・交差・重なりを防止
  - 各手順書の例 XML（②③④）を新ルール準拠の座標・接続点に差し替え

### Added
- `harness/_mece_guide.md` に対応策の軸「達成手段で分ける（自前／既存サービス・製品／外注）」と、失敗例「品質を犠牲にした安直案で止まらない」を追加
- `templates/02_原因ツリー_why.md` に「根本原因の一覧（全件）」表、`templates/03_対応策ツリー_how.md` に「カバレッジ表」と「目指す品質水準」列、`templates/04_アクションプラン.md` に品質目標・品質合格ラインの列を追加

### Removed
- 「真因を1〜3個に絞る」ステップ（②）と、「すぐ着手＝クイックウィン」を優先する考え方（③④）
