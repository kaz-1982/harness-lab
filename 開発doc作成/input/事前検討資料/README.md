# 事前検討資料の投入先(Phase 0 / 任意)

このフォルダは、**要件定義に入る前に「問題・原因・対応策」を検討した資料**を投入する場所です。

## 使い方

1. 既にある検討資料(企画メモ、課題整理、As-Is 分析、提案書、議事録 など)を、**このフォルダに置く**だけです。
2. ハーネスが認識する許可形式(v0.9〜): `.md` / `.txt` / `.docx` / `.pptx` / `.xlsx` / `.pdf` / `.png` / `.jpg` / `.jpeg` / `.gif` / `.svg`。
3. その状態でハーネス(ClaudeCode)を起動すると、**インテークの前(Phase 0)で自動的に読み込み**、内容を「問題 → 原因 → 対応策 → 想定要件・制約・前提」に正規化して `output/00_事前検討/PR-1_事前検討サマリ.md` を生成します。
4. 生成されたサマリの要約が提示されるので、認識合わせ(訂正・補足)をしてください。その内容が以降のインテーク・要件定義の入力シードになります。

## 資料が無い場合

**資料が無くても問題ありません。** このフォルダに許可形式のファイルが1件もなければ Phase 0 はスキップされ、通常どおりインテークの質問から始まります。

## 注意

- この `README.md` は説明ファイルなので、ハーネスは検討資料として扱いません(読み込み対象から除外されます)。
- 以下の OS / エディタ生成ファイルもトリガ対象外: `.DS_Store`, `Thumbs.db`, `desktop.ini`, `~$xxx.docx`, `*.tmp`, `*.swp`, `*.bak`, `.gitkeep`, `.keep`。これらが残っていても Phase 0 は誤発火しません(v0.9〜)。
- 上記の許可形式に該当しない資料(`.key`, `.numbers`, `.odt`, 音声・動画 等)はテキスト化 or PDF 化して再投入してください。それが難しい場合はハーネスから要点の口頭補足を求められます(推測で埋めないため)。
- 機密情報を含む資料を置く場合は、案件のセキュリティポリシーに従ってください。
