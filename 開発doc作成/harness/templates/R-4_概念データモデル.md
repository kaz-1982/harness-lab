# {{案件名}} - 概念データモデル

## 1. 目的
業務で扱う主要な概念(エンティティ)と関係を、実装に依存しない高い抽象度で整理する。
後続の B-12 論理データモデル / DM-1 ドメインモデルの出発点とする。

## 2. 適用範囲・スコープ
- 含むもの: 主要エンティティ(名詞抽出)とその関係、必要最小限の主要属性
- 含まないもの: 物理設計(型・桁・インデックス)、正規化の詳細(B-12 / D-8 で扱う)

## 3. 前提・依存
- 参照ドキュメント: R-1(業務要件定義書)、R-13(用語集 / 用語整合のため)
- 採番済ID: 主要エンティティに `E-*` を採番し `_id_registry.md` に登録する(B-12 / B-13 / DM-1 で同一の `E-*` を流用し二重定義しない)

|---|---|---|
| {{E-001}} | {{顧客}} | {{サービスを利用する個人・法人}} |
