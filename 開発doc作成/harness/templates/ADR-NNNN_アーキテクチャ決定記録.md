> # ADR-0001: ドメイン層をフレームワーク非依存にする
>
> **ステータス**: Accepted
>
> **コンテキスト**: 基幹システムの想定寿命は10年以上で、その間に Web フレームワークの世代交代が見込まれる。ビジネスロジックがフレームワークに密結合すると移行コストが高騰する。
>
> **決定**: レイヤードアーキテクチャを採用し、Domain 層は標準ライブラリのみに依存させる。永続化・Web は Infrastructure / Presentation 層に隔離する。
>
> **検討した選択肢**: (A) フレームワーク密結合 — 開発は速いが寿命リスク大 / (B) レイヤード+Domain隔離 — 初期コスト中・長期保守性高 / (C) フルヘキサゴナル — 柔軟だが小規模には過剰。
>
> **影響**: ポジティブ=フレームワーク更新の影響を Domain 層から遮断。ネガティブ=層間のマッピング実装が増える。後続=DM-1 ドメインモデルを Domain 層の設計根拠とする。
