## セッション要約(2026-06-26)

v0.11 までで以下が完了済(本 `harness-v0.8-improvements` ブランチ 4 コミット):
- v0.8: WORKFLOW 分離 + 必須テンプレ事前同梱 + check.py 拡張(孤立 / AC-AT / TBD)
- v0.9: depends_on 検証 + ファイル名整合 + ADR 突合 + Phase 0 allowlist + CLAUDE.md スリム + pre-commit
- v0.10: 用語整合 + 循環検出 + golden sample + 本体/案件物理分離
- v0.11: セッション引き継ぎ + テンプレ DSL + 英語対応の第一歩

ユーザーとの議論で確定した v0.12 の方向性:

### 確定事項
- **要件・設計 repo と 実装 repo は引き続き「分ける」** 方針を維持
- **同期方式**: `git submodule`(ローカルパス)を採用
  - Windows / macOS 両対応必須
  - 一人 + 一台 + バージョン管理したいユースケース
  - 設計 repo を実装 repo の `docs/design/` に submodule として配置
- **ADR を両 repo の同期キー**として扱う(実装中の設計変更は ADR で記録 → 両 repo に反映)
- **影響レベル分類**(議論で合意済):
  - 小: 設計影響なし → 実装 repo のみ
  - 中: 設計影響あり → ADR を切る → 後で D-* / B-* 更新
  - 大: 要件レベル → 実装一時停止 → 設計 repo に戻る

### 保留 / 不採用
- **英語対応の継続展開**(残り 10 本のテンプレ英訳、CLAUDE_en.md 等)
  - ユーザーが英語不要のため低優先度。v0.11 で入れた英語基盤(spec + 3 本)は寝かせる
- **`check.py --code-sync`**(実装コードと設計書の命名整合検証)
  - 言語依存で重く、ROI が不明確なため見送り候補
