# セッション引き継ぎログ(v0.11〜)

> このファイルは **セッション境界で Claude が追記** します。
> 規模の大きい案件をセッション分割で進める際の「再開点」と「意思決定の経緯」を残す目的です。
> 詳細は [WORKFLOW.md §9](../WORKFLOW.md)(セッション分割の運用ガイド)を参照。

## 使い方

- **セッション終了直前**: 直近セッションで完了したドキュメント / 進行中の作業 / 次の開始点 / 意思決定メモ / 残 TBD / `check.py` 結果 を構造化して追記する。`python harness/tools/check.py --session-end output` でテンプレを stdout に出力できる(コピペして編集)。
- **セッション開始時**: 末尾のエントリを読み、続きから着手すべきポイントをユーザーに提示してから作業を始める。`python harness/tools/check.py --session-start output` で末尾エントリを表示できる。
- ADR を切るほどではない「設計上の小さな判断」も「意思決定メモ」に書いておくと、後で「なぜそうしたか」を思い出せる。
