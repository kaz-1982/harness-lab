# ClaudeDesign 引き渡しパッケージ

このディレクトリは、ハーネス (Phase 1) から ClaudeDesign (Phase 2) への画面設計依頼パッケージです。

全体フロー: [../../WORKFLOW.md](../../WORKFLOW.md)

---

## 目的

ClaudeDesign に画面設計を依頼するために必要な情報を、このディレクトリ単独で完結するように集約しています。
ClaudeDesign 側からは元のハーネスを参照できないため、必要な資料はすべてこのディレクトリ内にコピーで配置します。

---

## ClaudeDesign 側で作成してほしい成果物

ハーネス側に戻すべき成果物は以下:

1. **B-7 画面一覧・画面遷移図** (`B-7_画面一覧_画面遷移図.md`)
   - 全画面のID(S-XXX)と名称
   - 画面遷移図(`.drawio.svg` 形式を正本とし `diagrams/` 配下に配置。Mermaid は補助)
   - 各画面の起動条件・呼び出し元

2. **B-8 画面設計書** (`B-8_画面設計書.md`)
   - 各画面のレイアウト
   - 画面項目(項目名・型・必須/任意・桁数・初期値・バリデーション)
   - 画面操作(ボタン・リンク → 動作)
   - 画面が呼び出す機能ID(F-*)・API ID(A-*)
   - エンティティとの対応(B-12 論理ER参照)
   - アクセス権限(ロール別表示・操作可否)

3. **画面モック / プロトタイプ**(任意)
   - `mockups/` 配下に画像 or Figma 等のリンク

4. **取り込みログ** (`_import_log.md`)
   - 作成日 / バージョン / ClaudeDesign プロジェクトURL
   - ハーネス側の引き渡しパッケージのバージョン(参照したドキュメントの版)

## 守ってほしい規約

ハーネス側で整合性を保つため、ClaudeDesign で作業する際は以下に従ってください:

- **用語**: `refs/R-13_用語集.md` に登録された用語のみを使用。新規用語が必要な場合は理由を記録(取り込み時に追加登録)
- **エンティティ名**: `refs/B-12_論理ER図.md` のエンティティ名と完全一致させる
- **ID採番**:
  - 画面ID は `S-001` から連番(ゼロパディング3桁)
  - 機能ID(F-*)、API ID(A-*)が新規発生したら `_import_log.md` に列挙(取り込み時にハーネス側 `_id_registry.md` に登録)
- **アクセス権限**: `refs/R-9_非機能要件定義書.md` のロール定義に準拠
- **非機能要件**: 応答時間・同時アクセス数等は `refs/R-9_非機能要件定義書.md` を上限に設計
