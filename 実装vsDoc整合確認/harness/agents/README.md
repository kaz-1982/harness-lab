# エージェント・ロスター(並行サブエージェント)

逆同期を**サブエージェントの並行実行**で高速化するための役割定義集。各ファイルは Claude Code のカスタムエージェント定義(frontmatter `name`/`description`/`tools` + システムプロンプト)としてそのまま `.claude/agents/` に置けるし、オーケストレータ(親 Claude)が `Agent`/`Task` を spawn する際のプロンプト雛形としても使える。

> 思想: **役割は汎用ハーネスに常設し、案件固有(APP_DIR・DOC_DIR・担当ID範囲)はプロンプト引数で注入**する。これで再利用性と速度を両立する。

## ロスター

| 役割 | ファイル | 種別 | 並列の境界 | 書き込み |
|---|---|---|---|---|
| 実装の棚卸し | `code-inventory.md` | 読み取り専用 | 1体(または領域別に複数) | なし |
| docの棚卸し | `doc-inventory.md` | 読み取り専用 | 1体(または領域別に複数) | なし |
| ドリフト検出 | `drift-detector.md` | 読み取り専用 | **ID種別ごと**(A-/T-/F-/UT- …) | なし |
| ドキュメント追従 | `doc-reconciler.md` | 読み書き | **担当docごと(重複不可)** | 担当docのみ |

## 守るべき3つの制約(暴走防止)

これを破ると並列化が事故になる。プロンプトに必ず含める:

1. **書き込みは1ファイル1オーナー**。`_id_registry.md` / `_doc_plan.md` / `_review_log.md` などの共有台帳は **親(オーケストレータ)だけが更新**する。サブには触らせない。検出系は read-only なので何体でも安全に並列、追従系は**担当docを重複させない**。
2. **各サブは自己完結の仕様で渡す**。サブエージェントは記憶を共有しない。毎回プロンプトに「APP_DIR / DOC_DIR / 担当範囲 / コードを正 / 推測禁止 / 最小差分 / 確認済みの実装事実」を同梱する。
3. **検証は親で集約**。`check.py` 緑・ID整合・「直さなかった記録」の確認は分散させず親が行う。

## 親(オーケストレータ)の責務

- 入力確定(APP_DIR/DOC_DIR)・正の原則合意・トリアージ(R1)
- 検出系をファンアウト → 戻りをマージして `_drift_report.md` を確定
- **ユーザー判断(方式差など)を仰ぐ**
- 新規IDの**一括採番**(`_id_registry.md`)
- 追従系をファンアウト(担当doc割当) → 戻りを受けて検証(R5)
